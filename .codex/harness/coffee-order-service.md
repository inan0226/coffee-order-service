# 커피 주문 서비스 하네스

이 문서는 구현 중 요구사항에서 벗어나는 실수를 줄이기 위한 작업 하네스다.

## 시작 체크리스트

- 어떤 요구사항을 변경하거나 구현하는지 확인한다.
- 해당 작업에 맞는 스킬 문서를 읽는다.
- 구조를 제안하기 전에 기존 코드를 먼저 확인한다.
- `build.gradle`을 수정하기 전에 의존성 추가가 정말 필요한지 판단한다.

## 계획 템플릿

```text
요구사항:
영향받는 API:
도메인 규칙:
저장 방식 필요 여부:
실패 케이스:
추가할 테스트:
```

## 생성 템플릿

```text
1. domain/model 코드 추가 또는 수정
2. repository/storage 코드 추가 또는 수정
3. service 코드 추가 또는 수정
4. controller 및 DTO 코드 추가 또는 수정
5. 테스트 추가 또는 수정
```

## 평가 템플릿

```text
집중 테스트:
전체 테스트:
요구사항 체크리스트:
남은 위험:
```

## 중단하고 사용자에게 확인할 조건

아래 상황에서는 계속 진행하기 전에 사용자에게 확인한다.

- 구현이 시작된 뒤 저장 방식 자체를 바꾸려는 경우
- 프로젝트 패키지명을 전체 변경하려는 경우
- Redis, Kafka, Docker, 외부 결제 API처럼 과제에 필요하지 않은 인프라를 추가하려는 경우
- 파괴적인 git 또는 파일 시스템 작업을 하려는 경우
