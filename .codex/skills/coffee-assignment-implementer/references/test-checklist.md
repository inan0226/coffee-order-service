# 테스트 체크리스트

## 메뉴 API

- 초기 메뉴 전체를 반환한다.
- 응답에 메뉴 ID, 이름, 가격이 포함된다.

## 포인트 충전

- 신규 사용자의 포인트를 충전한다.
- 기존 사용자에게 추가 충전하면 잔액이 누적된다.
- 0 이하의 충전 금액은 거절한다.

## 주문 및 결제

- 포인트가 충분하면 주문을 생성한다.
- 주문 성공 시 메뉴 가격만큼 포인트를 차감한다.
- 주문 성공 시 Mock 주문 이벤트를 1회 전송한다.
- 포인트 부족 주문은 실패한다.
- 포인트 부족 실패 후 포인트가 차감되지 않는다.
- 실패한 주문은 Mock 이벤트를 전송하지 않는다.
- 존재하지 않는 메뉴 ID는 거절한다.

## 인기 메뉴

- 성공 주문을 메뉴별로 정확히 집계한다.
- 인기 메뉴는 최대 3개만 반환한다.
- 7일보다 오래된 주문은 제외한다.
- 주문 수가 같은 경우 정렬 기준이 결정적이다.

## 최종 검증

- 변경 코드에 대한 집중 테스트를 실행한다.
- Windows에서는 최종 전달 전 `.\gradlew.bat test`를 실행한다.
