# 도메인 규칙

## 메뉴

- 메뉴는 ID, 이름, 가격을 가진다.
- 가격은 양의 정수여야 한다.
- 관리자 API가 없다면 과제 검증을 위해 작은 메뉴 목록을 초기 데이터로 넣는다.

## 포인트

- 사용자는 포인트 잔액을 가진다.
- 포인트 충전은 요청 금액만큼 잔액을 증가시킨다.
- 충전 금액은 양수여야 한다.
- 잔액은 절대 음수가 되면 안 된다.

## 주문

- 주문은 메뉴가 존재하고 사용자가 충분한 포인트를 가진 경우에만 성공한다.
- 결제와 주문 생성은 원자적으로 처리한다.
- 성공 응답을 반환하기 전에 포인트를 차감한다.
- 인기 메뉴 집계를 위해 주문 시각을 저장하거나 기록한다.
- 시간 범위 테스트가 가능하도록 주문 시각은 주입 가능한 `Clock`을 사용하는 편이 좋다.

## 데이터 수집 플랫폼 Mock

- 외부 플랫폼은 `OrderEventClient` 같은 interface 뒤에 둔다.
- 전송 데이터에는 사용자 식별값, 메뉴 ID, 결제 금액이 포함되어야 한다.
- 테스트에서는 fake 또는 mock으로 대체하고, 성공 시 1회 호출되는지 검증한다.
- 주문 생성이 실패하면 호출하지 않는다.

## 인기 메뉴

- 현재 시점 기준 최근 7일 안에 생성된 성공 주문만 집계한다.
- 주문 횟수 내림차순으로 정렬한다.
- 주문 횟수가 같으면 메뉴 ID 오름차순처럼 결정적인 tie-breaker를 둔다.
- 최대 3개 메뉴만 반환한다.
