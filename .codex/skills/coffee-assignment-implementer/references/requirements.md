# 과제 요구사항

## 1. 커피 메뉴 목록 조회 API

- 메뉴 ID, 이름, 가격을 반환한다.
- 응답은 검증하기 쉽고 항상 같은 형태여야 한다.

## 2. 포인트 충전 API

- 결제는 포인트로만 가능하다.
- 사용자 식별값과 충전 금액을 입력받는다.
- 충전 금액은 `1원 = 1P`로 계산한다.
- 0 이하의 충전 금액은 거절한다.
- 충전 후 잔액 또는 현재 잔액을 반환한다.

## 3. 커피 주문 및 결제 API

- 사용자 식별값과 메뉴 ID를 입력받는다.
- 메뉴 가격을 조회한다.
- 사용자의 충전 포인트에서 메뉴 가격을 차감한다.
- 포인트가 부족하면 주문을 거절한다.
- 성공한 주문 내역을 기록한다.
- 사용자 식별값, 메뉴 ID, 결제 금액을 데이터 수집 플랫폼 Mock으로 실시간 전송한다.

## 4. 인기 메뉴 목록 조회 API

- 최근 7일간 인기 있는 메뉴 3개를 반환한다.
- 메뉴별 주문 횟수 기준으로 집계한다.
- 성공한 주문만 집계한다.
- 7일 범위를 벗어난 주문은 제외한다.
- 응답에는 메뉴 ID, 이름, 가격, 주문 횟수를 포함한다.
