# Spring API 테스트 점검 스킬

## 테스트 전략

비즈니스 규칙은 빠른 service 테스트를 우선하고, JSON API 계약이 중요한 부분은 controller 테스트로 검증한다.

## 필수 테스트 그룹

1. 메뉴 목록
   - 메뉴 ID, 이름, 가격이 반환되는지 검증한다.

2. 포인트 충전
   - 신규 사용자 충전.
   - 기존 사용자 추가 충전.
   - 잘못된 충전 금액 거절.

3. 주문 결제
   - 성공 시 주문 생성.
   - 성공 시 포인트 차감.
   - 성공 시 Mock 주문 이벤트 정확히 1회 전송.
   - 포인트 부족 시 실패.
   - 실패 시 포인트 미차감.
   - 실패 시 이벤트 미전송.
   - 존재하지 않는 메뉴 실패.

4. 인기 메뉴
   - 메뉴별 주문 수를 정확히 집계.
   - 최근 7일 주문만 포함.
   - 최대 3개 반환.
   - 동률 정렬 기준이 결정적임.

## 평가 규칙

- Spring context 로딩 테스트만으로는 충분하지 않다.
- 최근 7일 테스트에서 실제 현재 시간에 의존하지 않는다. 시간을 주입하거나 제어한다.
- 테스트 실행 순서에 의존하지 않는다.
- 테스트 메서드명은 검증하는 규칙을 분명히 드러낸다.

## 완료 게이트

최종 응답 전 아래 항목을 확인한다.

```text
집중 테스트 통과:
전체 테스트 통과:
테스트로 검증하지 못한 요구사항:
검증하지 못했다면 이유:
```
